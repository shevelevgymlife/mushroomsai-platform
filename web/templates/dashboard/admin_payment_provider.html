Пусто при сохранении станет «как в Environment» (YOOKASSA_TELEGRAM_RECEIPT_VAT_CODE / YOOKASSA_RECEIPT_VAT_CODE). 0 — не передавать чек в счёте (если магазин без фискализации в Telegram). 1 — типично без НДС (самозанятые); точное значение — по справочнику ЮKassa. Удобно, если бот на отдельном сервисе Render без тех же переменных окружения, что и сайт.

Вместе с секретом ниже включает оплату на сайте и в Telegram Mini App (переход на страницу ЮKassa). Для пробного второго магазина (тестовые ключи) можно не менять эти поля, а задать в Render: YOOKASSA_OVERRIDE_SHOP_ID и YOOKASSA_OVERRIDE_SECRET_KEY — тогда API и вебхук используют их (боевые в БД остаются).

Что от вас нужно

  1. ЮKassa: shopId и секретный ключ (уже есть личный кабинет).
  2. BotFather: подключить к этому же боту платежи ЮKassa и скопировать provider token сюда.
  3. В ЮKassa в HTTP-уведомлениях указать URL выше (после деплоя с HTTPS).
  4. Тарифы и цены — в Тарифы подписок (блок ниже на этой странице).
